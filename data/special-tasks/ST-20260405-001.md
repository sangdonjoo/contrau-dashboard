## Big Picture

짜빈 미세조류(Spirulina) 공장의 실시간 모니터링 대시보드를 만들기 위한 첫 단계.
현재 데이터 수집이 파편화된 구글시트로 흩어져 있다.

**이번 ST의 목표:** 공장 전 팀이 매일 정해진 시점에 데이터를 입력하고, 대시보드 data-status 화면에서 수집 현황이 실시간 확인되는 상태를 만든다.

**수집 방식:** Zalo OA 대화형 — 정해진 시간에 팀장에게 푸시 → 자연어 응답 → AI 파싱 → DB 저장.
이상한 값이 들어오면 AI가 반문해서 교정.

**최종 성공 조건:** data-status 페이지에서 1일 전체 수집율 100%를 1회 달성.

**이후 로드맵:** 이 인풋 데이터를 기반으로 각종 분석 대시보드 개발 (별도 ST).

### 배경 자료
- 대시보드 data-status 페이지: `src/app/spirulina/data-status/page.tsx`
- 현재 mock 데이터: `src/data/db-status-mock.ts` (8팀 38항목 — 가짜, 실제 조직에 맞게 재정의 필요)
- SSOT 생산 스펙: `contrau-ssot/05_production/08_spec/SPEC_PRODUCTION.md`
- 팜 운영 KB: `contrau-ssot/05_production/01_raw/algae/FARM_OPERATIONS_TRA_VINH.md`
- 기존 Google Apps: `contrau-ssot/05_production/apps/algae/` (App1-App7)

## Steps

### Step 1 — 팀/데이터/담당자/시점 전수 정의 + 기존 시트 제출
- **담당:** charlie (git: nguyenvancu@192.168.68.118)
- **설명:** 짜빈 공장의 전체 팀 구조, 각 팀장, 팀별로 매일 수집해야 하는 데이터 항목, 수집 횟수(1일 1회/2회/이벤트별), 수집 시점(몇 시)을 빠짐없이 정의한다. 기존에 사용하던 모든 구글시트 URL을 제출하고, 정의한 데이터 항목이 해당 시트에서 실제로 확인되어야 한다.
- **측정:** 정의된 팀 수, 항목 수, 제출된 시트 수. 모든 항목에 대해 시트에서 최근 실제 수치가 확인됨.
- **완료기준:** (1) 전체 팀×항목 정의 완료 (2) 모든 구글시트 URL 제출 (3) 각 항목의 최근 실데이터가 시트에서 확인됨. 빠진 항목이나 불명확한 정보가 없는 상태.
- **상태:** prepared
- **진행율:** 0%

#### Step 1 인터뷰 가이드

이 단계는 **다문다답 딥인터뷰**로 진행한다. 한 턴에 여러 질문을 보내고 한꺼번에 답변받는다.

**1차 질문 묶음 — 팀 구조 + 데이터 전수 조사:**
- 짜빈 공장에 현재 몇 개 팀이 있나요? 각 팀 이름과 팀장(이름+역할)을 모두 알려주세요.
- 각 팀별로 매일 수집하는 데이터 항목을 전부 나열해 주세요. (항목 이름, 단위, 수집 횟수/일, 수집 시점)
- 하루 2회 이상 수집하는 항목이 있으면, 각 회차의 구체적 시각을 알려주세요. (예: OD 1/2 = 07:00, OD 2/2 = 16:00)
- 이벤트 기반(수시) 수집 항목이 있으면 어떤 이벤트 발생 시 기록하는지 알려주세요.
- 현재 데이터 기록에 사용하는 모든 구글시트 URL을 제출해 주세요.

**2차 질문 묶음 — 검증 + 보완** (1차 답변 기반):
- 제출된 시트에서 각 항목의 최근 실데이터를 확인한다.
- 시트에 없는 항목이나 불명확한 정보가 있으면 추가 질문한다.
- 시트 간 중복/누락을 정리하여 최종 확정한다.

**완료 판정:** 전체 팀×항목 매트릭스가 완성되고, 모든 항목에 대해 시트에서 실데이터가 확인되면 AI가 완료 판정.

### Step 2 — DB 구성 + Zalo OA 수집 + 대시보드 실데이터 표시
- **담당:** sangdon (git: sangdon.joo@contrau.eco)
- **설명:** Step 1에서 확정된 팀/항목/시점을 기반으로: (1) Supabase 수집 테이블 생성 (2) Zalo OA에서 각 팀장에게 정해진 시점에 수집 메시지 발송 (3) 자연어 응답 AI 파싱 → DB 저장 (4) data-status 페이지를 mock에서 실데이터로 전환하여 수집 현황 표시.
- **측정:** data-status 페이지에서 실데이터로 표시되는 항목 비율 (0% → 100%).
- **완료기준:** 전체 항목이 Zalo OA로 수집되어 DB에 저장되고, data-status 페이지에서 실데이터 기반으로 표시됨.
- **상태:** prepared
- **진행율:** 0%

#### Step 2 실행 가이드

1. Step 1 결과(팀×항목 매트릭스)를 읽고 Supabase 스키마 설계
2. `db-status-mock.ts`의 mock 데이터를 실제 팀/항목 구조로 교체
3. Zalo OA 수집 봇 구현: 스케줄러 → 팀장별 푸시 → 응답 파싱 → DB insert
4. data-status 페이지를 Supabase 실데이터 기반으로 전환
5. 전체 항목이 실데이터로 표시되는지 확인

**완료 판정:** data-status 페이지에서 모든 팀의 모든 항목이 실데이터(Supabase) 기반으로 렌더링되면 AI가 완료 판정.

### Step 3 — 대시보드 피드백
- **담당:** charlie (git: nguyenvancu@192.168.68.118)
- **설명:** 구현된 data-status 화면과 실제 수집 현황을 확인하고 피드백한다. 항목명, 단위, 표시 방식, 누락, 오류 등. 피드백이 없어도 "확인 완료, 이상 없음" 피드백을 제출해야 한다.
- **측정:** 피드백 항목 수 (0건이면 "이상 없음" 1건 카운트).
- **완료기준:** 찰리가 피드백을 제출함.
- **상태:** prepared
- **진행율:** 0%

#### Step 3 인터뷰 가이드

**다문다답 1회:**
- 대시보드 data-status 페이지를 확인해 주세요. (URL 제공)
- 팀 이름, 담당자, 항목명이 맞나요? 틀린 것이 있으면 모두 알려주세요.
- 수집 시점, 횟수가 실제 운영과 맞나요?
- 표시 방식(색상, 배치, 단위 등)에 대해 바꾸고 싶은 것이 있나요?
- 빠진 항목이나 추가하고 싶은 항목이 있나요?
- 기타 의견이 있으면 자유롭게 알려주세요.

**완료 판정:** 찰리가 피드백(또는 "이상 없음")을 제출하면 완료.

#### Step 4 실행 가이드

**완료 판정:** AI가 모든 피드백 항목의 반영을 확인하면 완료.

### Step 5 — 일일 수집 100% 달성

- **상태:** prepared
- **진행율:** 0%

**완료 판정:** data-status 페이지에서 특정 일자의 수집율이 100%로 확인되면 AI가 완료 판정. ST 전체 완료.
